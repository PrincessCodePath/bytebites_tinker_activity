# ByteBites Tinker

## What This Was

For this activity we had to design the backend structure for a campus food ordering app called ByteBites. The goal was to take a short feature request and turn it into a working system. We first identified four main classes that the app needed, created a UML-style diagram with help from AI, and then turned that design into Python code and added additional features. This assignment helped me practice using AI as a tool during the process while still making the final decisions ourselves.

## Where AI Helped and Where It Did Not

AI was most helpful when reviewing the code. Sometimes it suggested small improvements for readability or structure. One thing that surprised me was that sometimes it simply said the code looked clear and did not need changes. At first I expected it to always suggest something, but when it did that it was helpful because it confirmed that the implementation was good and that it was not suggesting unnecessary additions.

Where AI was less helpful was at the beginning when I asked it to generate the UML diagram and class structure, because it tended to add extra classes and features that were not specified. I had to adjust my prompts, and once I narrowed the instructions, the outputs became much closer to what I intended.

## How I Would Help a Student

If another student were stuck, I would suggest that they look back at the spec and the UML diagram and explain what each class is supposed to represent in their own words. Usually if someone can explain the role of each class, almost like pseudocode, the code becomes easier to write.

Instead of giving them the exact solution, I would encourage them to work through one small step at a time. I would also suggest using AI to ask specific questions, such as understanding an error message or thinking through how a method should behave.

## Where Students Might Struggle

One challenge is figuring out how to ask the AI the right questions. If the prompt is too vague, the AI may add features that were never requested. Students might also struggle with running the tests and interpreting error messages. When a test fails, they need to read the output carefully to understand what was expected and whether the issue is in the code or the test itself.
